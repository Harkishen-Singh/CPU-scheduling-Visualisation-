CPU Sheduling Algorithms

These are set of protocols or rules, by which a process is executed in a specific manner. A typical process involves both I/O time and CPU time. In a uniprogramming system like MS-DOS, time spent waiting for I/O is wasted and CPU is free during this time. In multiprogramming systems, one process can use CPU while another is waiting for I/O. This is possible only with process scheduling.
Some important terms :

  1. Arrival Time: Time at which the process arrives in the ready queue.
  2. Completion Time: Time at which process completes its execution.
  3. Burst Time: Time required by a process for CPU execution.
  4. Turn Around Time: Time Difference between completion time and arrival time.
    Turn Around Time = Completion Time - Arrival Time
  5. Waiting Time(W.T): Time Difference between turn around time and burst time.
    Waiting Time = Turn Around Time - Burst Time
